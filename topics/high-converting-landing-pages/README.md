## Question
How should an indie software product design a landing page that converts both
individual users and enterprise buyers?

## Why it matters
Informs the marketing-site design for Ishoo (and other indie products) where a
single page must serve self-serve users and enterprise decision makers at once.

## Key questions / sub-threads
- [x] One message, dual-path, or two sites?
- [x] What must the first screen / above-the-fold answer?
- [x] Imagery, social proof, and CTA patterns that convert.

## Current state
Full landing-page design research report mined from the `ishoo` repo.
Recommendation: a **dual-path page** — one obvious self-serve path for individuals
and a clearly available enterprise path — with a first screen that answers what,
who, what outcome, why credible, and what to do next. See findings.
